# 候选 idea（idea-generation 阶段产出）

> 针对 gap「重采样系统性破坏概率校准」,多角度发散(gap-driven / problem-reframe / method-transfer / efficiency),再二维收敛。
> 算力前提:本机 CPU 16 核,sklearn 1.5,目标"快而严谨"。数据用 UCI/OpenML 真实公开不平衡数据集。

## 发散候选

### A（gap-driven，稳妥）⭐ 收敛首选
**一句话**:系统量化 SMOTE/随机过采样/欠采样对树集成(RF/GBDT)**概率校准**的破坏,并检验事后校准(Platt/isotonic)能否在不损 AUC 下修复。
- **创新点**:相对 SMOTE 家族(`10.1613/jair.953`、综述 `10.1613/jair.1.11192`)只评 F1/AUC,本工作把**校准误差(ECE/Brier/可靠性曲线)**作为一等结果,across 多数据集多种子做严谨实证。
- **凭什么更强(机理假设)**:重采样改变训练先验 π,使输出概率系统性偏离真实后验;**竞争性解释**:也可能是树集成本身在不平衡下的叶节点概率估计偏差,需用"重采样 vs 仅改类权重"分离两者。
- **解决什么(可证伪)**:H1 重采样显著抬高 ECE(配对检验 p<0.05);H2 事后校准能把 ECE 拉回接近原始且 AUC 损失<0.005。
- **可行性**:UCI/OpenML 不平衡数据集(如 credit-g 1000×20、Pima 768、成人收入子集);sklearn RF/HistGBDT + CalibratedClassifierCV;CPU 几小时可完成多种子。
- **风险(精确IF)**:若某数据集本就接近平衡(IR<2),重采样效应不显著 → 故意选 IR 跨度大的数据集集合规避。
- **成果**:IEEE 会议/期刊短文。**投稿层次**:稳妥(实证严谨型)。

### B（problem-reframe，高风险高回报）
把"重采样要不要做"重构成"在期望代价/阈值决策下,重采样+校准 vs 仅代价敏感学习 谁更优"。风险:范围易膨胀,时间紧。

### C（efficiency，保底）
只做"重采样幅度(采样比 ρ)× 校准损失"的单数据集精细扫描曲线。保底但贡献单薄。
