# rubric.md — 八维度 behavioral anchors + 权重 + 加权聚合 + decision mapping

本文件是 idea 严审的**可量化评分卡**。每个验收维度给出 5 个分段（90–100 / 75–89 / 60–74 / 45–59 / <45）的**具体证据形态**（behavioral anchor，描述"打到这一档要看到什么"，而非空泛形容词），并附权重。聚合公式与判决映射见末尾。

锚定金标准：NeurIPS/ICLR 评审表（Originality/Quality/Significance + Soundness/Presentation/Contribution）、ScholarEval 五点量表、ResearchAgent 的 novelty/clarity/validity。维度不自创——是把顶会评审表展开成 idea 阶段可操作的证据形态。

## 0. 评分前提（读不到这些 → 该维度封顶 60，并在判决里标 "evidence-missing"）
- 新颖性维度：必须有**真实检索结果**（OpenAlex/Semantic Scholar/arXiv 至少一库，记 HTTP 码与最像的 3 篇），否则封顶。
- 数据维度：必须能说出**数据集名称 + 规模 + 标注方式**，"应该够"=红旗，封顶 60。
- 实验维度：必须能写出**至少一组干净对照 + 一项消融**，写不出封顶 60。
凡用 "should / probably / 应该 / 大概" 支撑某维度评分 → 该维度自动降一档并记红旗。

## 维度 1：创新性 / Originality（权重 0.20）
衡量相对已发表/被拒工作的真实 delta，必须基于检索而非记忆。

- **90–100**：检索到的最像 3 篇与本 idea 有**可命名的机理级差异**（不只是换数据集/换 backbone）；能指出前作的具体失败模式并说明本 idea 为何能解；delta 足以独立支撑一篇顶会主结果。
- **75–89**：与最像工作有清晰差异点（新问题设定 / 新机制 / 新约束），但 delta 偏增量；检索覆盖到位，confidence 标注合理。
- **60–74**：差异存在但偏工程组合（A+B 拼接、已知模块换序）；检索做了但只覆盖单库，可能高估新颖性。
- **45–59**：基本是已有工作换数据集/换指标重跑，或"首次把 X 用到 Y"但 X、Y 都成熟且无非平凡适配。
- **<45**：套壳/概念堆叠；检索即可找到几乎同构的已发表工作；或宣称新颖但**未做任何检索**。

## 维度 2：理论深度 / Soundness（权重 0.18）
有无机理/假设/可解释性，主结论是否真从证据推出。

- **90–100**：有明确可证伪假设 + 机理解释，且能预先说明"什么结果会推翻它"；因果主张配有混杂控制设计。
- **75–89**：有合理机理叙述与假设，逻辑链完整，但部分环节靠经验而非推导。
- **60–74**：以直觉/类比驱动，假设隐含未明述；结论多为相关性却措辞偏因果。
- **45–59**：纯试错堆叠，无机理；主结论存在逻辑断链（即便数据成立也推不出）。
- **<45**：地基假设可证伪或无据；自相矛盾。

## 维度 3：数据支撑 / Data（权重 0.14）
现有数据能否验证 idea，规模/质量/标注是否够。

- **90–100**：能点名具体数据集 + 规模 + 标注方式，规模与标注足以支撑主张与消融；已识别已知偏倚/许可风险（联动 m02/db04）。
- **75–89**：数据可得且基本够用，但某一维度（规模/标注质量/分布覆盖）有可控缺口，有补救方案。
- **60–74**：数据存在但需大量清洗/再标注，或规模勉强；缺口未量化。
- **45–59**：依赖尚不存在或需自建的数据，可行性未论证；只说"数据应该够"。
- **<45**：无可用数据，或验证 idea 所需数据在现实约束下拿不到。

## 维度 4：实验可控性 / Quality（权重 0.14）
能否设计干净对照与消融，证明增益来自创新点本身而非工程 trick。

- **90–100**：能写出干净对照 + 逐组件消融 + 可证伪的 negative control，能预先说明"如果增益来自 trick 会怎样表现"。
- **75–89**：有合理对照与主消融，但未覆盖全部混杂因素（如算力/数据量/超参不对齐）。
- **60–74**：对照可设但消融粗糙，难以归因增益来源。
- **45–59**：对照不公平（baseline 弱化/超参未对齐）或无消融。
- **<45**：无法设计能分离创新点贡献的实验；增益与工程 trick 无法区分。

## 维度 5：方法贡献 / Contribution（权重 0.13）
贡献是否清晰、可量化、可复现。

- **90–100**：贡献一句话可述、可量化（指标 + 预期幅度）、可复现（方法步骤清晰）；非平凡。
- **75–89**：贡献清晰可量化，复现路径基本明确，存在少量未定细节。
- **60–74**：贡献可识别但量化模糊（"提升性能"无幅度）或复现需要补大量细节。
- **45–59**：贡献泛泛，与已有工作贡献高度重叠。
- **<45**：说不清贡献，或贡献为已知常识的复述。

## 维度 6：差异化 / Significance-delta（权重 0.08）
相对已发表/被拒工作的 delta 是否足以支撑一篇论文/一个奖项。

- **90–100**：拉到同主题真实 review（OpenReview）佐证，本 idea 恰好解决了审稿人挑过的痛点；delta 显著。
- **75–89**：相对同类工作有清晰增量贡献，够一篇扎实论文。
- **60–74**：delta 存在但偏边缘，可能被审稿人评为 incremental。
- **45–59**：与近期工作高度同质，难以说清"为什么需要这一篇"。
- **<45**：已被同类工作覆盖或拒稿过的设定，无新增。

## 维度 7：可行性 / Feasibility（权重 0.07）
算力/时间/工程难度是否现实。

- **90–100**：所需算力/时间/工程量明确且在现实约束内，有降级方案。
- **75–89**：基本可行，存在可控的工程/算力压力。
- **60–74**：可行性偏紧，关键资源（算力/标注/工程）有不确定性。
- **45–59**：需要稀缺资源或超长周期，未论证如何获得。
- **<45**：现实约束下无法完成。

## 维度 8：影响力 / Impact（权重 0.06）
问题多重要，谁会引用/采用。

- **90–100**：解决领域公认痛点，明确的受众会引用/采用，可能开新方向。
- **75–89**：对一个活跃子领域有实在价值。
- **60–74**：价值局限于窄场景。
- **45–59**：问题重要性存疑或受众模糊。
- **<45**：问题不重要或已被解决。

## 加权聚合公式

每维度取 0–100 的分值 `s_i`，权重 `w_i`（见各维度，合计 = 1.00）：

```
Weighted = Σ (s_i × w_i)        # 0–100 连续分
```

权重表（合计 1.00）：

| 维度 | 权重 |
|---|---|
| 1 创新性 | 0.20 |
| 2 理论深度 | 0.18 |
| 3 数据支撑 | 0.14 |
| 4 实验可控性 | 0.14 |
| 5 方法贡献 | 0.13 |
| 6 差异化 | 0.08 |
| 7 可行性 | 0.07 |
| 8 影响力 | 0.06 |

**否决项优先于加权分**（gate before score）：以下任一成立，无论 Weighted 多高，最高只能"有条件通过"或直接"不通过"——
- 创新性 < 45 → 直接不通过（套壳/无检索）。
- 任一 Devil's Advocate CRITICAL 未化解 → 最高"有条件通过"。
- 创新性、理论深度、数据、实验 四个核心维度中**有两个 < 45** → 不通过。

## Weighted → Overall（1–10）映射

便于和 references.md 第 1 条 NeurIPS Overall 量纲对齐：

```
Overall ≈ round(1 + Weighted / 100 * 9)   # 线性映射到 1–10
```

## decision mapping 表（Overall + 否决项 → 判决）

| Weighted | Overall | 默认判决 | 条件 |
|---|---|---|---|
| ≥ 80 | 7–10 | **通过** | 且无未化解 CRITICAL、无核心维度 <60 |
| 65–79 | 6 | **有条件通过** | 列 Revision Roadmap，补强 <60 的维度 |
| 50–64 | 5 | **有条件通过（重大）** | 等价 major revision，可能需回 m03 改设定 |
| 35–49 | 4 | **不通过** | 给 ≥3 个具体改进方向，回 m03 |
| < 35 | ≤3 | **不通过** | 地基问题，建议换 idea |

判决以"否决项 + 此表"二者取**更严格**的结果。Confidence 低（检索不足/领域陌生）时不给极端分，并在判决里显式标注 confidence 与 evidence gaps。
