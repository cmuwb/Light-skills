# Light 9 个数据库的性质归类与处置方案

> 分析日期 2026-06-13 ｜ 方法：9 个库各用一个 Explore agent 读 README + 抽样真实卡（不是 README 宣称），按"内容性质"归类估占比，判断换学科用户是否被误导、是否会过时，给砍/留/转实时建议。这是只读分析，未改动任何库。
>
> 触发问题：用户问"本地数据库会不会太局限/太少/给不同方向用户用出问题"，甚至"要不要干脆别要本地库"。本文用真实占比回答：**砍多少、留什么、为什么。**

---

## 一、核心结论

**9 个库根本不是一类东西。过去用同一套"静态卡 + 月度保鲜"对待它们，是病根。**

按真实内容性质，每个库的内容可拆成四类：

- **A-通用判断**：跨方向通用的方法论/经验/判断（摘要五段式、y 轴不能截断造假、权利要求独立项求最大保护范围、答辩 10-15 页）。对任何学科都成立，**几乎不过时**，是 OpenAlex 等在线源永远给不了的——**这是 Light 真正的护城河，该留下精养。**
- **A-偏科判断**：带方向前提的判断（"SSD 已过时"只在 CV 时间线成立、"奶山羊白羊 re-id 区分度低"只在该畜牧场景成立）。换学科**会误导**，解法是**按领域打标签隔离**，不是砍。
- **B-易变事实**：高频变动的客观事实（期刊 IF/分区/APC、数据集下载链接/许可/被引数、代表作）。在线源（OpenAlex/Crossref/HuggingFace）**更新更快更全更准**，本地副本必过时——**该转实时查。**
- **C-项目状态**：项目自身的记忆（项目卡/决策日志/版本历史）。与学科无关，**必须本地、不可转实时**。

**一句话**：不能全砍，也不能全留。按库分别处置——把该转的转（B）、该留的留（A-通用）、该隔离的隔离（A-偏科）、该锁本地的锁本地（C）。

## 三、逐库处置方案

### 主体转实时（事实副本，维护已跟不上）

**db01 期刊会议（~80% 转）**
- 现状：308 venue 行，本质是 OpenAlex Sources + Crossref + CCF 目录 + 出版社官网的离线快照。impact_factor 列用 OpenAlex 2yr 均被引代理（非真 JCR IF），1798 处"待核查"，107 行代表作空着，全部统一 2026-06 一个时间戳（无增量真实性）。
- 转：IF/分区/APC/被引/works_count/模板链接/投稿链接 → 按需实时查 OpenAlex Sources / Crossref / DOAJ / 出版社官网。venue_signal.py 已是这个路子，推广到全部事实字段。
- 留（~5%）：reference_style 格式映射（IEEE/ACM/LNCS 多年不变）、ai_policy 出版社级政策（变动慢、查一次复用）、README 的采集核验方法论（双源核验铁律 / 19 列 schema / CSV 转义规则）——这是"怎么去在线源建表"的方法，是真资产。
- 隔离（~15%）：CCF 等级、subject_area、review_cycle 估值——CCF 是中国 CS 专属，须标"仅 CS"并与在线分区隔离，否则材料/医学用户被误导。

**db04 数据集（~55% 转）**
- 现状：~100 卡严重集中在 CV/多模态/LLM/动物视觉（约 76/100）。citation 是 OpenAlex 被引快照（带 2026-06-06 日期、自承"随时间增长"）。
- 转：被引/license/download_url/paper_url/leaderboard_url → 实时查 OpenAlex/Crossref/HuggingFace/PapersWithCode。
- 留（~15%）：preprocessing_steps/recommended_splits 里的通用方法论（"按患者不重叠划分防泄漏""time-forward rolling split"）抽出来上提为跨学科"数据使用方法论"层。
- 隔离（~25%）：逐数据集 bias_risk/known_issues（"KITTI 仅德国单城晴天""ImageNet 西方中心"）须按领域标，不能当通用资产。
- 本地（~3%）：奶山羊自建数据集决策/现状评估卡（项目记忆性质）。

### 拆开处置

**db03 方法（留 40% / 转 30% / 隔离 25%）**
- 留：core_assumption/advantages/limitations/common_baselines/evaluation_metrics 是跨方向方法学知识（DFT 基态假设、知识蒸馏暗知识、SHAP Shapley 公理化），提炼为通用方法论层。
- 转：representative_papers 的 cited:/doi:/year（246 条被引快照）、implementation_repo 的 GitHub 状态、suitable_datasets → OpenAlex/Crossref/GitHub/HuggingFace 实时查。
- 隔离（最严重）：176 卡里 82 卡（47%）是奶山羊/家畜 CV，maturity 标签（"SSD 过时""NeRF 被 3DGS 替代"）只在 CV 时间线成立。须标"仅本项目/CV 域适用"，不混进通用方法卡。

### 主体留（真资产，只剥易变事实）

**db02 写作（留 55%）**：patterns_library.md 全部（摘要五段、IMRaD、hedging、措辞红线、reviewer 追问）跨学科通用、引 Whitesides/PLOS 经典不过时。转：被引/DOI/venue/CCF 等级（36 处快照）。隔离：24 张样本卡 20+ 张是 AI/CV/ML（ResNet/VGG/VAR），"竞赛排名最硬/开源做 social proof/beat baseline"是 AI 文化，套到无 leaderboard 的社科/理论会写歪——须打领域标签并按用户学科补非 AI 范本。

**db05 前端设计（留 58%）**：玻璃拟态/新拟物的可访问性逻辑、WCAG、栅格、字体层次——学科中立。转：resources_real Part1 的 26 条工具的许可/定价/链接（shadcn MIT、MUI X 收费等，本库已自标"待核查"）→ 实时查官方 LICENSE/官网。偏科仅表层措辞（suitable_project_scene 举例科研化），不构成误导。

**db06 PPT（留 65%）**：一页一观点/action-title/data-ink/答辩控时/海报字号——跨学科通用，不像"YOLO 改 backbone"那种偏科。转：resources_real 15 条资源的 star/许可/状态（reveal.js 71.6k★ 等）→ GitHub/PyPI API。本地：themes.py 10 主题调色板（已 selftest）。

**db07 图表（留 60%）**：y 轴不截断、气泡面积∝数值非半径、雷达图轴顺序统一——制图通识不过时。转：工具被引/DOI/gallery URL（Matplotlib 39283 等）。隔离：volcano/Manhattan/KM/CONSORT/Bland-Altman 等领域专属图型按 research_field 标（已有该字段）。本地：self_produced 9 卡指向项目自家 g1-g9。

**db08 软著专利（留 62%）**：与"CV 方法库"相反的典型——软著登记/专利撰写/大创挑战杯申报在中国是**全学科共用的行政程序**，材料化学生和社科生走同一套结构（权利要求独立项/从属项、软著源代码连续性、TAM-SAM-SOM）。几乎不偏科（~3%）。转：检索平台 API 现状（PatentSearch API 需 key、EPO OPS OAuth）、竞赛官方 URL、加急费用、评分权重——指向官方源 + 待核查，不缓存数值。

### 必须本地

**db09 项目状态（留 ≥90%）**：82% 是 project_card/decision_log/version_history/saved_search——在线源没有"你这个项目上周决定弃用 re-id"这种数据，按定义必须本地持久。其中嵌入的 B-fact（IF/被引/许可，~5%）坚持现有纪律：只存快照 + 来源指针，回指 db01/db04 与在线源，需最新值时实时查而非信本地副本。lessons.md 的跨项目方法论（~8%）天然已分离。

## 四、"不同方向用户被误导"是独立的第三个问题

用户最初担心"本地库给不同方向用户用会出问题"——数据证明这是**真问题，但来源是 A-偏科，不是"本地"本身**：

- **不偏科的库**（db08 行政程序、db05/db06 设计版式、db02 写作方法论的 patterns 部分）：换学科照用，不误导。
- **重偏科的库**（db03 方法 47% 奶山羊CV、db04 数据集 76% CV、db02 样本卡 AI 范本）：换学科会被无关内容淹没或被 CV 时间线判断误导。

**解法不是砍，是隔离**：给偏科卡强制打 `research_field` / `domain_scope` 标签（db07 已有 research_field 字段，是正面样板），让非该方向用户能过滤；通用判断层抽出来，对谁都成立。

> 全砍本地库 = Light 退化成"提示词 + 实时上网"，与 gpt-researcher / 直接用 Claude 无异。任务2 横向分析已证：Light 唯一压过高 star 通用 agent 的，正是这些沉淀的跨学科通用判断。砍光 = 自废护城河。

## 五、落地优先级（供后续动手重构）

> 本文只做分析。真要重构时建议顺序：

**P0（止血，影响诚实可信）**
- db01/db04 的 B-事实字段改"实时查 + 薄缓存（只存快照 + last_checked + 来源指针）"，对齐 venue_signal.py 已有模式；停止维护静态全量快照。
- 给所有"信本地卡"的技能加规则：**本地快照与在线源冲突时，标注两值 + 默认信在线**，消除"看起来权威实则过时"的假确定性。

**P1（隔离偏科，影响跨学科可用）**
- db03/db04/db02 的偏科卡强制打 `domain_scope` 标签，奶山羊/CV 专属内容隔离到项目层或显式标"仅 X 域"。
- 通用判断层（各库 A-通用）抽出来集中，作为对任意学科成立的方法论资产。

**P2（精养真资产）**
- db02/05/06/07/08 的方法论卡按学科补非 AI/非科研味范本，扩通用性。
- freshness 检查从 warn-only 提为对 B-缓存的强制时效门（超期即标，不静默输出旧值）。
