## 3. 三类历史风险（写任何新内容前自查）

审计证实，本仓库最大风险不是"内容错"，而是：
1. **跨技能口径分裂**——同一事实两套说法（OpenAlex 限流、Science 栏宽都栽过）。→ 新写一个事实前，先 grep 全仓库有没有别处也在说它；有就指认单一真相源，别处只放指针。
2. **单向挂载**——A 声明 B 会消费，B 不知道（orchestrator 契约、m08 findings 都栽过）。→ 凡写"X 交给 Y"，必须同轮去 Y 里写上对应的消费/产出声明。
3. **安装视角断链**——仓库内有效、装完失效（CONVENTIONS 引用栽过）。→ 凡新增跨目录引用，自问"装到 ~/.claude/skills 后这个相对路径还通吗"。

## 5. 每轮固定收尾八步（硬性）

1. 全量门禁（在 `<LIGHT_ROOT>` 下）：
   ```bash
   python tools/check_skills.py && \
   python tools/check_entry_docs.py && \
   python tools/check_skill_links.py && \
   python tools/check_skill_assets.py && \
   python tools/check_installation_assets.py && \
   python tools/check_databases.py && \
   python tools/run_skill_selftests.py
   ```
2. `git status --short` 必须干净（selftest/演示不许留产物）；`git diff --check` 无空白错误。
3. 本轮验收清单逐条核对（在各轮文件末尾），未达项写入 PROGRESS 偏差日志。
4. 提交：**按用途分组多个 commit**；标题中文、"范围:内容"用途口吻（参考：`记忆管理技能:沉淀Hermes会话链恢复流程`）；**不带任何 AI 署名**（无 Co-Authored-By）。
5. 推送并确认 CI 绿（docs/ 不触发 CI 属正常；技能/库改动必须等绿）。
6. 更新 [PROGRESS.md](PROGRESS.md)：状态、commit 区间、偏差、留给下轮的话。
7. references 类改动补"研究日期：2026-06-XX"标头；新脚本登记 WHATS_INCLUDED.md；新 mode 登记 MODE_REGISTRY.md；路由改动同步 ROUTER_EXAMPLES.md。
8. 向用户打印下一轮的新对话启动提示词（从 11 号文件复制并按 PROGRESS 现状微调）。
