# Light 路由样例回归集

这张表是 `ROUTER.md` 的人工可读测试集，也是 `.github/scripts/check_entry_docs.py` 的机读输入。目标是防止入口文档漂移：用户输入相似但意图不同，应该落到稳定的技能组合。

- `m##` / `a##` 代码以 `ROUTER.md` 为准。
- `light-orchestrator` 只在跨阶段 pipeline 或断点恢复/接手上下文时出现。
- 常驻技能可作为辅助路由列出，但主阶段技能要放在前面。

| 用户输入 | 期望主路由 | 说明 |
|---|---|---|
| `帮我把这个方向做一遍文献调研` | `m01` | 文献调研/综述入口 |
| `这个方向有哪些经典工作和最新工作？` | `m01` | 方向扫描与代表作 |
| `帮我清洗这个 CSV，看缺失值和异常值` | `m02` | 数据质量与清洗 |
| `这个数据够不够支撑论文实验？` | `m02` | 数据可用性评估 |
| `给我想几个有创新点的研究 idea` | `m03` | idea 生成 |
| `这个 idea 够不够创新，帮我严格挑刺` | `m04` | idea 严审 |
| `我怀疑你审 idea 太严了，做一下校准` | `m04` | idea-critique calibration 子模式 |
| `把这个 idea 变成完整研究方案和实验矩阵` | `m05` | 研究方案/实验设计 |
| `实验跑完了，这些结果说明什么？` | `m06` | 结果解释 |
| `帮我做显著性检验、效应量和置信区间` | `m06` | 统计分析 |
| `论文里应该做哪些图、放在哪一节？` | `m09` | 图表规划，不直接画图 |
| `把这些结果画成出版级组图` | `m11` | 实际绘图 |
| `这张图的配色和双 y 轴有没有问题？` | `m11` | 图表诚实性/绘图质量审查 |
| `帮我写摘要和引言初稿` | `m07` | 论文起草 |
| `把方法章节重写一下，逻辑更清楚` | `m07` | drafting 的 section-redraft |
| `帮我对这份草稿做失败模式自检` | `m07` | paper-drafting 的草稿 self-review，不等同于 m14 正式模拟审稿 |
| `这段论文语言不通顺，帮我润色` | `m08` | 语言/结构润色 |
| `帮我核验这些 DOI 并生成 BibTeX` | `m10` | 引用核验/格式 |
| `这个 LaTeX 编译 log 报错，帮我定位` | `m12` | 排版/编译日志 |
| `这篇论文该投哪个期刊，冲稳保怎么选？` | `m13` | 投稿定位 |
| `模拟三个审稿人帮我预审这篇论文` | `m14` | 投稿前模拟审稿 |
| `我收到审稿意见了，帮我写 response letter` | `m14` | 真实返修回复 |
| `帮我准备软著功能说明和源码材料` | `m15` | 软著材料 |
| `帮我把这个技术方案写成专利交底书` | `m15` | 专利交底/权利要求辅助 |
| `帮我做一套答辩 PPT` | `m16` | 幻灯片制作 |
| `帮我检查这套 PPT 每页视觉是否统一` | `m16` | deck 视觉 QA |
| `互联网+ 项目申报书和路演材料怎么写？` | `m17` | 竞赛/申报材料 |
| `读一下这个 PDF 讲了什么` | `a01` | 文件读取 |
| `这个 Excel 里有哪些 sheet 和字段？` | `a01` | 结构化文件读取 |
| `记住这个项目当前进展，下次继续` | `a02` | 记忆/项目管理 |
| `我们上次做到哪了？` | `a02` | 项目状态恢复 |
| `继续，刚断了` | `a02` `light-orchestrator` | 断点恢复/跨轮续接，先读状态再继续 |
| `接手 Claude 之前做的这个技能包优化` | `a02` `light-orchestrator` | 接手上下文与跨阶段维护 |
| `继续写刚才那一段 discussion` | `m07` | 只是续写文本，不启动 orchestrator |
| `继续润色上一段` | `m08` | 只是单段润色，不启动 orchestrator |
| `从文献调研到实验设计帮我全流程规划` | `light-orchestrator` `m01` `m05` | ≥3 阶段 pipeline |
| `写实验代码把 baseline 跑起来` | `a03` | 后端/实验代码 |
| `设计数据库表、接口和权限` | `a04` | 系统设计 |
| `做一个可视化大屏前端原型` | `a05` | 前端/可视化平台 |
| `把这个项目目录整理成可复现结构` | `a06` | 项目结构 |
| `论文和 PPT 里的指标名称不一致，帮我统一` | `a07` | 跨材料一致性 |
| `交付前帮我整体自审一遍` | `a08` | 自我审查 |
| `这件事该用 Python、R 还是 MATLAB？` | `a09` | 工具选择 |
| `这个研究有没有伦理或隐私风险？` | `a10` | 科研伦理/合规 |
